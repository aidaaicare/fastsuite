Programming defaults

OLP


In the Programming defaults dashboard all the base attributes for the project set-up and for the technology are specified. Each parameter can be set with a default value that will be taken into account when programming the toolpaths.


It is highly recommended to define all settings in the Programming defaults before generating the tool path, because not all properties or attributes will dynamically update an existing tool path after they have been modified in this dashboard.

General

This container opens the general set up of the project.


Attribute

Description

Remark

Tool Frame

The coordinate system of the resource that is going to run the tool path.


Base Frame

The reference coordinate system to which the tool path coordinates are calculated when the program is computed for download.

Technology

The manufacturing technology package that will be applied when computing the tool path and operations.

The attribute value is empty when no technology has been assigned.

Automatic tool change

To enable to program operations with intermediate tool change(s).

Optional.

The active tool frame and base frame from the OLP setup are displayed here. When needed the drop down list of available frames can be opened and another frame can be set active.

Work methods

This container opens the defaults for the different work methods that the technology includes. A work method can be seen as a strategy of how certain input is being used and / or reworked to program a toolpath.

For each technology the content of this container is different. But in general, opening the container shows a horizontal list of available work methods. Each of them will have its own container(s) with attributes that can be opened to set the default values for that work method. The picture below shows an example.
