Controller geometry

Controller Builder


The Geometry command is a helpful, but not mandatory, command to maintain the geometry of the resource for a realistic 3D representation of that resource. When designing the resource it is used in combination with the Import command.

After import, the geometry is visible, but has not become part of the resource yet. Saving the resource at this stage and closing the document will loose the geometry.

To have the resource represented with the imported geometry and to be able to have the geometry react correspondingly with the resource's motions, the geometry has to be attached to the joints and frames of the resource definition. In any workbench or function within the software, the imported geometry is displayed normally, with its original or manipulated colors. When this command is active the display shows which geometry is or is not already attached to the resource. Attached geometry is given a new temporary color, non-attached geometry is displayed in a light gray shaded mode of the original color.
